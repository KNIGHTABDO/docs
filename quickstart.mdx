# Quickstart

Get ZeroQCM up and running locally in minutes. This guide covers:
- Cloning and installing the project
- Configuring Supabase credentials
- Starting the development server
- Navigating the platform

<Note>
This quickstart gets you running for local development. For production deployment on Vercel, see the [Installation Guide](/installation).
</Note>

## Step 1: Clone and install

```bash
git clone https://github.com/KNIGHTABDO/zeroqcm.git
cd zeroqcm
npm install
```

<Note>
  Requires **Node.js 18+** and **npm 9+**.
</Note>

## Step 2: Configure environment

Copy the example environment file and fill in your Supabase credentials:

```bash
cp .env.example .env.local
```

At minimum, set these variables in `.env.local`:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
```

<Tip>
  Get your Supabase URL and anon key from **Supabase Dashboard → Settings → API**.
</Tip>

## Step 3: Start the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser. You should see the ZeroQCM landing page.

## Step 4: Explore the platform

<Tabs>
<Tab title="Dashboard">
  After signing in, the **Dashboard** shows your selected semester with modules and activities. Pick a module to start a quiz session.
</Tab>

<Tab title="Quiz Engine">
  The quiz interface supports:
  - **Keyboard navigation**: `A`–`E` to select options, `Enter` to confirm, `←/↑` to navigate
  - **Swipe gestures**: Swipe left/right on mobile
  - **Instant feedback**: Correct/incorrect answers highlighted immediately
</Tab>

<Tab title="AI Explanations">
  During a quiz, tap any option to get an AI-generated explanation of why it is correct or incorrect. Responses stream token-by-token.
</Tab>

<Tab title="Stats & Progress">
  The **Stats** page shows animated SVG rings for total answered, success rate, and daily streak. The **Revision** page targets your weakest modules.
</Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
<Accordion title="Dev server won't start">
  Check that:
  - Node.js version is 18 or higher: `node --version`
  - Dependencies are installed: `npm install`
  - Port 3000 is not in use
</Accordion>

<Accordion title="Supabase connection errors">
  - Verify `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` in `.env.local`
  - Confirm your Supabase project is active in the dashboard
  - Check that Row Level Security (RLS) policies are configured
</Accordion>

<Accordion title="Auth not working">
  - Ensure Supabase Auth is enabled with email provider
  - Check the redirect URL configuration in Supabase Auth settings
  - Verify `NEXT_PUBLIC_APP_URL` matches your local URL
</Accordion>
</AccordionGroup>
