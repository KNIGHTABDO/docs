# Architecture

ZeroQCM is a **Next.js 15** application using the App Router, TypeScript, Supabase, and Tailwind CSS.

## Tech Stack

| Layer | Technology | Purpose |
|---|---|---|
| **Framework** | Next.js 15 (App Router) | SSR, routing, API routes |
| **Language** | TypeScript 5 | Type safety |
| **Database** | Supabase (PostgreSQL) | Data storage, auth, realtime |
| **Styling** | Tailwind CSS 3 | Utility-first CSS |
| **Animations** | Framer Motion | Page transitions, micro-interactions |
| **State** | Zustand | Client-side state management |
| **AI** | GitHub Models + AI SDK | Streaming AI explanations |
| **Auth** | Supabase Auth | Email/password authentication |
| **Deployment** | Vercel | Serverless hosting |

## Key Architectural Decisions

### App Router
ZeroQCM uses the Next.js **App Router** (not Pages Router) for:
- Server Components by default
- Nested layouts
- Server Actions
- Streaming responses

### Supabase for Everything
Supabase provides database, auth, and realtime in a single service:
- **PostgreSQL** with Row Level Security
- **Auth** with JWT session management
- **Realtime** for study rooms and live features
- **Storage** for user avatars (if applicable)

### Client-Side State with Zustand
Zustand manages transient client state like:
- Selected semester
- Quiz navigation state
- Theme preferences

### Design Tokens
All colors use CSS custom properties defined in `globals.css`:
- No hard-coded color values in components
- Dark/light mode via CSS variable swapping
- Consistent theming across the application
