# Development Setup

This guide helps you set up a local development environment for contributing to ZeroQCM.

## Prerequisites

- **Node.js 18+** and **npm 9+**
- A [Supabase](https://supabase.com) account
- A [GitHub](https://github.com) account
- Git

## Getting Started

<Steps>
  <Step title="Fork the repository">
    Fork [KNIGHTABDO/zeroqcm](https://github.com/KNIGHTABDO/zeroqcm) to your GitHub account.
  </Step>

  <Step title="Clone your fork">
    ```bash
    git clone https://github.com/YOUR_USERNAME/zeroqcm.git
    cd zeroqcm
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment">
    ```bash
    cp .env.example .env.local
    ```

    Fill in your Supabase credentials. See [Configuration](/hosting/configuration) for details.
  </Step>

  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000).
  </Step>
</Steps>

## Available Scripts

| Command | Description |
|---|---|
| `npm run dev` | Start dev server with Turbopack |
| `npm run build` | Create production build |
| `npm run lint` | Run ESLint |
| `npm run lint:fix` | Auto-fix lint issues |
| `npm run typecheck` | Run TypeScript type checking |

## Making Changes

```bash
# Create a feature branch
git checkout -b feature/your-feature

# Make changes and test locally
npm run dev

# Run linting and type checking
npm run lint
npm run typecheck

# Commit and push
git commit -m "feat: description of your change"
git push origin feature/your-feature
```

Then open a pull request on GitHub.
