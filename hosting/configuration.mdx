# Configuration

ZeroQCM is configured through environment variables in a `.env.local` file (local development) or Vercel environment variables (production).

## Environment Variables Reference

### Supabase (Required)

<ParamField path="NEXT_PUBLIC_SUPABASE_URL" type="string" required>
  Your Supabase project URL.

  ```bash
  NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
  ```

  Find this in **Supabase Dashboard → Settings → API**.
</ParamField>

<ParamField path="NEXT_PUBLIC_SUPABASE_ANON_KEY" type="string" required>
  Supabase anonymous (public) key for client-side access.

  ```bash
  NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIs...
  ```

  This key is safe to expose to the browser — Row Level Security (RLS) policies protect your data.
</ParamField>

<ParamField path="SUPABASE_SERVICE_ROLE_KEY" type="string">
  Supabase service role key for server-side operations with full database access.

  ```bash
  SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIs...
  ```

  <Warning>
    This key bypasses RLS. Never expose it to the client or prefix it with `NEXT_PUBLIC_`.
  </Warning>
</ParamField>

### AI Backend

<ParamField path="GITHUB_MODELS_TOKEN" type="string">
  GitHub Personal Access Token for the GitHub Models AI API. Used by the `/api/ai-explain` route.

  ```bash
  GITHUB_MODELS_TOKEN=ghp_your_personal_access_token
  ```

  <Note>
    Server-side only. Never exposed to the browser.
  </Note>
</ParamField>

<ParamField path="GITHUB_COPILOT_OAUTH_TOKEN" type="string">
  GitHub Copilot OAuth token for the internal Copilot API (optional fallback).

  ```bash
  GITHUB_COPILOT_OAUTH_TOKEN=ghu_your_oauth_token_here
  ```
</ParamField>

<ParamField path="GITHUB_COPILOT_CLIENT_ID" type="string">
  GitHub Copilot Device Flow client ID for acquiring OAuth tokens via the admin panel.

  ```bash
  GITHUB_COPILOT_CLIENT_ID=Iv1.b507a08c87ecfe98
  ```
</ParamField>

### Cron and Sync

<ParamField path="CRON_SECRET" type="string">
  Secret for authenticating the daily sync cron job at `/api/sync`.

  ```bash
  CRON_SECRET=your-random-secure-cron-secret
  ```

  Must match the `Authorization: Bearer` header configured in Vercel Cron.
</ParamField>

### App URL

<ParamField path="NEXT_PUBLIC_APP_URL" type="string">
  The public URL of the application.

  ```bash
  # Local development
  NEXT_PUBLIC_APP_URL=http://localhost:3000

  # Production
  NEXT_PUBLIC_APP_URL=https://zeroqcm.me
  ```
</ParamField>

## Example .env.local

```bash
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key

# AI
GITHUB_MODELS_TOKEN=ghp_your_personal_access_token
GITHUB_COPILOT_OAUTH_TOKEN=ghu_your_oauth_token_here
GITHUB_COPILOT_CLIENT_ID=Iv1.b507a08c87ecfe98

# Cron
CRON_SECRET=your-random-secure-cron-secret

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

## Security Notes

<Warning>
  - **Never commit `.env.local`** to version control — it is already in `.gitignore`
  - `NEXT_PUBLIC_` prefixed variables are exposed to the browser — only use for Supabase URL and anon key
  - `SUPABASE_SERVICE_ROLE_KEY` has full database access — keep it server-side only
  - `GITHUB_MODELS_TOKEN` is a server-side secret — never expose it to the client
</Warning>
