# Deployment Overview

ZeroQCM is designed to deploy on **Vercel** with automatic deploys from the `main` branch. The platform uses Next.js 15 with the App Router and Turbopack.

## Architecture

```
┌─────────────┐      ┌──────────────┐      ┌──────────────────┐
│   Browser    │ ───▶ │   Vercel     │ ───▶ │    Supabase      │
│   (React)    │ ◀─── │   (Next.js)  │ ◀─── │   (PostgreSQL)   │
└─────────────┘      └──────────────┘      └──────────────────┘
                            │
                            ▼
                     ┌──────────────┐
                     │ GitHub Models│
                     │   (AI API)   │
                     └──────────────┘
```

| Component | Service | Purpose |
|---|---|---|
| **Frontend** | Vercel (Next.js) | React UI with SSR and App Router |
| **API Routes** | Vercel Serverless | `/api/*` endpoints for AI, sync, and data |
| **Database** | Supabase | PostgreSQL with auth, realtime, and RLS |
| **AI Backend** | GitHub Models | GPT-4o, Llama 3.3, Phi-4, Mistral for explanations |
| **Auth** | Supabase Auth | Email/password authentication with JWT |

## Deploying to Vercel

<Steps>
  <Step title="Fork or clone the repo">
    Fork [KNIGHTABDO/zeroqcm](https://github.com/KNIGHTABDO/zeroqcm) or use your own clone.
  </Step>
  <Step title="Import to Vercel">
    Go to [vercel.com/new](https://vercel.com/new) and import the repository.
  </Step>
  <Step title="Add environment variables">
    Configure all required variables in **Vercel → Settings → Environment Variables**. See [Configuration](/hosting/configuration) for the full list.
  </Step>
  <Step title="Deploy">
    Click **Deploy**. Vercel builds and deploys automatically on every push to `main`.
  </Step>
</Steps>

## Vercel Settings

| Setting | Value | Why |
|---|---|---|
| Framework | Next.js | Auto-detected |
| Runtime for `/api/ai-explain` | `nodejs` | Edge Runtime can't read sensitive env vars |
| `maxDuration` on AI route | `60` | Prevents default 10s timeout from killing AI streams |
| Build Command | `next build` | Default |
| Output Directory | `.next` | Default |

## Custom Domain

Configure a custom domain in **Vercel → Project → Settings → Domains**:

1. Add your domain (e.g., `zeroqcm.me`)
2. Update DNS records as instructed by Vercel
3. SSL is provisioned automatically

## Cron Jobs

ZeroQCM uses Vercel Cron for daily data synchronization:

```json
// vercel.json
{
  "crons": [
    {
      "path": "/api/sync",
      "schedule": "0 2 * * *"
    }
  ]
}
```

The cron job runs at **02:00 UTC** daily and requires the `CRON_SECRET` environment variable for authentication.
