# Installation Guide

This guide covers the complete setup of ZeroQCM, from local development to production deployment on Vercel.

<Note>
  **Requirements**:
  - Node.js 18+ and npm 9+
  - A [Supabase](https://supabase.com) account (free tier works)
  - A [GitHub](https://github.com) account (for AI features)
  - A [Vercel](https://vercel.com) account (for deployment)
</Note>

## Local Development Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/KNIGHTABDO/zeroqcm.git
    cd zeroqcm
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment variables">
    ```bash
    cp .env.example .env.local
    ```

    Edit `.env.local` with your credentials. See [Configuration](/hosting/configuration) for the full variable reference.
  </Step>

  <Step title="Set up Supabase">
    1. Create a new project at [supabase.com](https://supabase.com)
    2. Run the database migrations (see [Database](/hosting/database))
    3. Copy the **URL** and **anon key** from **Settings → API**
  </Step>

  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    ZeroQCM starts at [http://localhost:3000](http://localhost:3000) using Next.js with Turbopack.
  </Step>
</Steps>

## Environment Variables

The minimum required variables for local development:

```bash
# Supabase (required)
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key

# GitHub Models — for AI explanations (optional for basic usage)
GITHUB_MODELS_TOKEN=ghp_your_personal_access_token

# App URL
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

<Tip>
  See the full environment variable reference in the [Configuration](/hosting/configuration) page.
</Tip>

## Vercel Deployment

ZeroQCM is designed to deploy on **Vercel** with automatic deploys from the `main` branch.

<Steps>
  <Step title="Import project">
    Go to [vercel.com/new](https://vercel.com/new) and import the `KNIGHTABDO/zeroqcm` repository.
  </Step>

  <Step title="Configure environment variables">
    Add all required environment variables in **Vercel → Project → Settings → Environment Variables**:
    - `NEXT_PUBLIC_SUPABASE_URL`
    - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
    - `SUPABASE_SERVICE_ROLE_KEY`
    - `GITHUB_MODELS_TOKEN`
    - `CRON_SECRET`
    - `NEXT_PUBLIC_APP_URL`
  </Step>

  <Step title="Deploy">
    Click **Deploy**. Vercel will build and deploy automatically.
  </Step>

  <Step title="Configure custom domain">
    In **Vercel → Project → Settings → Domains**, add your custom domain (e.g., `zeroqcm.me`).
  </Step>
</Steps>

### Critical Vercel Settings

| Setting | Value | Why |
|---|---|---|
| Runtime for `/api/ai-explain` | `nodejs` (not `edge`) | Edge Runtime can't read sensitive env vars |
| `maxDuration` on AI route | `60` | Prevents default 10s Hobby timeout from killing streams |
| `GITHUB_MODELS_TOKEN` | Encrypted env var | Server-side only, never exposed to browser |

<Warning>
  The `SUPABASE_SERVICE_ROLE_KEY` has full database access. Never expose it to the client — keep it as a server-side only environment variable.
</Warning>

## Building from Source

For custom builds or contributions:

```bash
# Install dependencies
npm install

# Run type checking
npm run typecheck

# Create production build
npm run build

# Start production server
npm run start
```
