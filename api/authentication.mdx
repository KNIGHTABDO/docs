# Authentication

ZeroQCM uses **Supabase Auth** with email/password authentication. Sessions are managed via JWT tokens stored in cookies.

## Auth Flow

<Steps>
  <Step title="Sign up or log in">
    Users authenticate at `/auth` using email and password. Supabase creates a session with access and refresh tokens.
  </Step>
  <Step title="Session stored in cookies">
    The Supabase SSR client (`@supabase/ssr`) stores session tokens in HTTP-only cookies for server-side access.
  </Step>
  <Step title="Middleware checks session">
    The Next.js middleware (`middleware.ts`) checks the Supabase session on every protected route request.
  </Step>
  <Step title="Redirect if unauthenticated">
    Unauthenticated users are redirected to `/auth` with a `redirect` query parameter preserving the original URL.
  </Step>
</Steps>

## Route Protection

The middleware categorizes routes into three groups:

### Public Routes
No authentication required:
- `/` — Landing page
- `/auth` — Login and signup
- `/api/*` — API routes (handle their own auth)
- `/_next/*` — Next.js static files
- `/favicon.ico`, `/og`, `/images/*`, `/icons/*`

### Protected Routes
Require authentication + activation:
- `/dashboard`, `/quiz`, `/semestres`, `/stats`
- `/revision`, `/bookmarks`, `/flashcards`
- `/chatwithai`, `/study-rooms`, `/voice`
- `/settings`, `/profil`, `/certificates`
- `/activate` (requires auth but not activation)

### Admin Routes
Require authentication + admin email:
- `/admin/*` — Admin panel and operations

## Activation System

After authentication, users must be **activated** before accessing protected routes:

1. New users are redirected to `/activate`
2. Users submit an activation key
3. An admin approves or rejects the key via the admin panel
4. Approved users (`activation_keys.status = 'approved'`) can access all protected routes

<Note>
  Admin users (identified by email) skip the activation check entirely.
</Note>

## Supabase Client Setup

ZeroQCM uses the `@supabase/ssr` package for server-side cookie management:

```typescript
import { createServerClient } from "@supabase/ssr";

const supabase = createServerClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
  {
    cookies: {
      getAll: () => request.cookies.getAll(),
      setAll: (cookiesToSet) =>
        cookiesToSet.forEach(({ name, value, options }) =>
          response.cookies.set(name, value, options)
        ),
    },
  }
);
```
