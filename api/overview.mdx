# API Overview

ZeroQCM uses **Next.js API routes** (App Router) for all server-side logic. Each route lives under `src/app/api/` and handles its own authentication.

## API Routes

| Route | Method | Description |
|---|---|---|
| `/api/ai-explain` | POST | Generate AI explanation for a quiz option |
| `/api/chat` | POST | ChatWithAI conversational endpoint |
| `/api/sync` | POST | Daily cron sync of question data |
| `/api/scrape-expand` | POST | Sync odd semester questions from DariQCM |
| `/api/scrape-even` | POST | Sync even semester questions from DariQCM |
| `/api/reset-stats` | POST | Reset all user statistics |
| `/api/weak-questions` | GET | Get user's weak questions |
| `/api/weak-quiz` | GET | Generate a targeted revision quiz |
| `/api/gh-models` | GET | List available GitHub Models |
| `/api/user` | GET/POST | User profile operations |
| `/api/activate` | POST | User activation key submission |
| `/api/admin/*` | Various | Admin panel operations |
| `/api/image-proxy` | GET | Proxy and cache external images |
| `/api/og` | GET | Generate Open Graph images |

## Authentication

API routes use two authentication strategies:

### User Authentication
Most routes check the Supabase session from cookies:

```typescript
const supabase = createServerClient(/* ... */);
const { data: { user } } = await supabase.auth.getUser();

if (!user) {
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}
```

### Cron Authentication
The `/api/sync` route uses a bearer token:

```typescript
const authHeader = request.headers.get("Authorization");
if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}
```

## Route Configuration

AI routes use special Vercel configuration for streaming:

```typescript
export const maxDuration = 60; // Prevent Hobby tier timeout
export const runtime = "nodejs"; // Not edge — needs env vars
```
